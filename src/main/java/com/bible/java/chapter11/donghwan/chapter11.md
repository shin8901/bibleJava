#Collection FrameWork
- 데이터 군을 저장하는 클래스들을 표준화한 설계(1.2부터)
- [Collection(List,Set), Map]
- 프로그래밍 실력을 더 갖추고 컬렉션 프레임워크의 소스를 분석해보자!!
- Vector, Stack, Hashtable, Properties와 같은 클래스들은 컬렉션프레임워크가 있기 전부터 있었기 때문에 명명법을 따르지않는다.
- Vector, Hashtable 보다는 ArrayList, HashMap을 사용하자
- Collections 클래스, Collection 인터페이스
 
#ArrayList
  - 접근시에 연속적으로 데이터가 있기 때문에  배열주소 + n*타입의크기 하면 바로 접근할 수 있다
  - 순차적으로 삭제시 수정/삭제시 빠르다
  
#LinkedList
  - 링크드 리스트는 불연속적으로 존재하는 데이터를 서로 연결한 형태로 구성
  - 다음 주소값을 가지고 있다
  - 이전 요소에 대한 접근이 어렵기 때문에 더블링크드 리스트로 이전 요소에 대한 값도 참조한다
  - 중간값을 추가/삭제시에 빠르다
  - 인덱스 접근시에 불연속적인 값을 찾아가야하기때문에 시간이 오래걸린다
  - 데이터가 많을수록 접근성이 떨어진다
