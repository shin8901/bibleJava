# 객체지향언어
1. 코드의 재사용성이 높다.
2. 코드의 관리가 용이하다.
3. 신뢰성이 높은 프로그래밍을 가능하게 한다.
<br> * 상속, 캡슐화 추상화 개념이 중심
<br> * 핵심: 코드의 재사용성이 높고 유지보수가 용이하다. + 코드의 중복제거

# 클래스
1. 정의: 객체를 정의해 놓은 것
2. 용도: 객체를 생성
<br> * 객체: 클래스에 정의된 내용대로 메모리에 생성 된 것을 뜻함
<br> * ex) 클래스: 붕어빵기계 / 객체: 붕어빵 

# 인스턴스
1. 정의: 클래스로부터 만들어진 객체를 그 클래스의 인스턴스라고 한다.
