#chaper11 InnerClass

```java
class A{
	class B{
	}
}
```

내부 인스턴스 변수처럼 사용되거나
선언된 영역에서만 사용되도록 제한한다거나,
관리 목적상 이너클래스로 관리하여, 연관관계를 높인다거나

다양한 사용이유가 있을수 있겟죠.

접근제어자나 제어자들을 통해서, 역시 static 하게 만들수도 있기도하고 final이 되기도함.

**익명클래스**
선언되고 바로 사용되어지는 일회성 클래스
이름이 없음

작성법은 AnonymousClass를 봅시다
